>  Moore 型和 Mealy 型自动机分别是什么？

可以这样记忆：
*   **Moore 机**：输出是状态的“名片”。你只要看到它当前在哪个状态，就知道它的输出是什么。
*   **Mealy 机**：输出是状态转换的“标签”。输出发生在**从 A 状态到 B 状态**的路上，并且取决于**你走哪条路（输入是什么）**。

下面我们来详细解释和对比。

---

### **Moore 型自动机**

在 Moore 模型中，**输出只与当前状态有关**。

*   **定义**：一个 Moore 机是一个六元组 `(Q, Σ, Λ, δ, γ, q0)`，其中：
    *   `Q`：有限状态集合
    *   `Σ`：有限输入字母表
    *   `Λ`：有限输出字母表
    *   `δ`：状态转移函数，`δ: Q × Σ → Q`
    *   ** `γ`：输出函数，`γ: Q → Λ` ** （**关键**：函数输入只有状态 `Q`）
    *   `q0`：初始状态

*   **工作原理**：
    1.  机器从初始状态 `q0` 开始，立即产生输出 `γ(q0)`。
    2.  读入一个输入符号 `a ∈ Σ`。
    3.  根据当前状态 `q` 和输入 `a`，通过 `δ(q, a)` 确定下一个状态 `q‘`。
    4.  进入新状态 `q‘` 后，**立即**产生与新状态对应的输出 `γ(q‘)`。
    5.  重复步骤 2-4。

*   **图形化表示（状态图）**：
    *   状态圈内标注为 ** `状态名 / 输出值` **。
    *   转移箭头只标注 ** `输入` **。
    *   **示例**：状态 `S1` 的输出是 `0`，则圈内写 `S1/0`。从 `S1` 到 `S2` 的转移条件是输入 `a`，则箭头上只标 `a`。

*   **特点**：
    *   输出比输入**延迟一个时钟周期**（在同步电路中）。因为输入导致状态改变后，新状态才产生新输出。
    *   输出稳定，只在状态改变时变化。

---

### **Mealy 型自动机**

在 Mealy 模型中，**输出与当前状态和当前输入都有关**。

*   **定义**：一个 Mealy 机也是一个六元组 `(Q, Σ, Λ, δ, λ, q0)`，其中：
    *   `Q, Σ, Λ, δ, q0` 的含义与 Moore 机相同。
    *   ** `λ`：输出函数，`λ: Q × Σ → Λ` ** （**关键**：函数输入是状态 `Q` 和输入 `Σ`）

*   **工作原理**：
    1.  机器从初始状态 `q0` 开始。**注意**：在初始时刻没有输入，通常认为 Mealy 机初始时没有输出（除非特别定义）。
    2.  读入一个输入符号 `a ∈ Σ`。
    3.  根据当前状态 `q` 和输入 `a`，**同时**做两件事：
        *   通过 `δ(q, a)` 确定下一个状态 `q‘`。
        *   通过 `λ(q, a)` 产生一个输出。
    4.  进入新状态 `q‘`。
    5.  重复步骤 2-4。

*   **图形化表示（状态图）**：
    *   状态圈内**只标注状态名**。
    *   转移箭头上标注 ** `输入 / 输出` **。
    *   **示例**：从状态 `S1` 到 `S2`，当输入为 `a` 时输出 `1`，则箭头上标 `a/1`。

*   **特点**：
    *   输出对输入**响应更快**，可以在同一个时钟周期内响应输入变化。
    *   输出可能更频繁地变化，因为即使状态不变，输入变化也可能导致输出变化。

---

### **核心对比表格**

| 特性 | Moore 型自动机 | Mealy 型自动机 |
| :--- | :--- | :--- |
| **输出依赖** | **仅依赖当前状态** | **依赖当前状态和当前输入** |
| **输出函数** | `γ: Q → Λ` | `λ: Q × Σ → Λ` |
| **状态图标注** | `状态 / 输出` | `状态` |
| **转移箭头标注** | `输入` | `输入 / 输出` |
| **输出时序** | 在状态**进入后**立即产生，**滞后**于输入 | 在状态转移**过程中**产生，**同步**或快于 Moore 机 |
| **响应速度** | 较慢（需先改变状态） | 较快（直接响应输入） |
| **状态数** | 实现相同功能，通常需要**更多状态** | 通常需要**更少状态** |
| **等价性** | 任何 Moore 机都可以转化为一个等价的 Mealy 机，反之亦然（输出时序上可能有细微差别）。 |

### **一个简单例子：检测输入序列中的 “01”**

**任务**：设计一个自动机，当检测到输入比特流中出现 “01” 子串时输出 1，否则输出 0。

#### **Mealy 机实现**
*   **状态**：`S0`（初始，未检测到有效模式），`S1`（刚收到一个 0）。
*   **转移**：
    *   在 `S0`：输入 0 -> 去 `S1`，输出 0；输入 1 -> 留在 `S0`，输出 0。
    *   在 `S1`：输入 0 -> 留在 `S1`，输出 0；**输入 1 -> 去 `S0`，输出 1**（检测到了“01”）。
*   **特点**：一旦在 `S1` 看到 `1`，**立刻**在转移边上输出 1。

#### **Moore 机实现**
*   **状态**：`S0/0`（初始），`S1/0`（刚收到 0），`S2/1`（已检测到“01”）。
*   **转移**：
    *   在 `S0/0`：输入 0 -> 去 `S1/0`；输入 1 -> 留在 `S0/0`。
    *   在 `S1/0`：输入 0 -> 留在 `S1/0`；输入 1 -> 去 ** `S2/1` **。
    *   在 `S2/1`：输入 0 -> 去 `S1/0`；输入 1 -> 去 `S0/0`。
*   **特点**：必须进入一个专门的输出状态 `S2/1` 才能输出 1，比 Mealy 机响应慢一拍。
