### 2. (简答题)

Suppose that coverage criterion C 1 subsumes【包含】 coverage criterion C 2. Further suppose that test set T 1 satisfies C 1 on program P and test set T 2 satisfies C 2, also on P.

(a) Does T 1 necessarily satisfy C 2? 

(b) Does T 2 necessarily satisfy C 1? 

(c) If P contains a fault, and T 2 reveals the fault, does T 1 necessarily also reveal the fault? Explain your answer.

(a) 基于包容的定义，因为准则 C 1 包含了准则 C 2, 根据定义，任何能够达到 C 1 100% 的测试集，也一定可以达到 C 2 的 100% 覆盖率。所以既然 T 1 满足 C 1, 它也一定满足 C 2  

(b) 满足较弱的测试准则 C 2 并不能保证可以满足更强的测试准则 C 1 . 比如说 C 1 是分支覆盖， C 2 是语句覆盖。 测试集 T 2 可能只是执行了所有的语句，但是并没有覆盖所有的真假分支情况  

(c) 不是必然的，覆盖率大并不能保证发现的缺陷多。虽然 T 1 满足了比 T 2 更严格的覆盖要求 C 1, 但是不以为测试集 T 1 是测试集 T 2 的超集。假如触发该缺陷需要一个非常特定的输入值，而 T 2 刚好有这个用例，那么就能发现缺陷。T 1 为了满足 C 1，执行更多的代码路径，但是如果输入并没有触发这个特定值，仍然发现不了缺陷

# 语句覆盖与分支覆盖详解

## 一、语句覆盖（Statement Coverage）

### 定义
> 语句覆盖要求测试用例能使程序中**每一条可执行语句**至少被执行一次。

### 计算公式
```
语句覆盖率 = (被执行的语句数 / 可执行语句总数) × 100%
```

### 示例

```python
def example(a, b):
    result = 0          # 语句1
    if a > 0:           # 语句2 (判断)
        result = a      # 语句3
    if b > 0:           # 语句4 (判断)
        result += b     # 语句5
    return result       # 语句6
```

**只需一个测试用例即可达到 100% 语句覆盖：**
| 测试用例 | a | b | 覆盖语句 |
|---------|---|---|---------|
| TC 1 | 1 | 1 | 1,2,3,4,5,6 ✅ 全部覆盖 |

> ⚠️ **问题**：`a ≤ 0` 或 `b ≤ 0` 的分支（即 `if` 为 `false` 的路径）从未被测试！

---

## 二、分支覆盖（Branch Coverage）

### 定义
> 分支覆盖要求测试用例能使程序中**每个判定的每个分支（True / False）** 至少被执行一次。

也称为 **判定覆盖（Decision Coverage）**。

### 计算公式
```
分支覆盖率 = (被执行的分支数 / 分支总数) × 100%
```

### 示例（同一段代码）

```python
def example(a, b):
    result = 0        
    if a > 0:           # 判定1: 有 True分支 和 False分支
        result = a    
    if b > 0:           # 判定2: 有 True分支 和 False分支
        result += b   
    return result     
```

**分支总数 = 4（2 个判定 × 2 个分支）**

| 测试用例 | a | b | 判定 1 | 判定 2 | 覆盖分支 |
|---------|---|---|-------|-------|---------|
| TC 1 | 1 | 1 | **T** | **T** | 判定 1-True, 判定 2-True |
| TC 2 | -1 | -1 | **F** | **F** | 判定 1-False, 判定 2-False |

> ✅ 两个测试用例覆盖了所有 4 个分支，达到 **100% 分支覆盖**。

## 三、两者关系 🔗

### 1. 包含关系（核心结论）

```
┌─────────────────────────────────┐
│        分支覆盖 (更强)           │
│   ┌─────────────────────────┐   │
│   │     语句覆盖 (更弱)      │   │
│   └─────────────────────────┘   │
└─────────────────────────────────┘
```

> **100% 分支覆盖 → 一定满足 100% 语句覆盖**
> **100% 语句覆盖 → 不一定满足 100% 分支覆盖**

### 2. 为什么分支覆盖能包含语句覆盖？

```python
if condition:
    statementA   # True 分支中的语句
else:
    statementB   # False 分支中的语句
```

- 分支覆盖要求 `True` 和 `False` 都走到 → `statementA` 和 `statementB` **都会被执行** → 语句自然被覆盖。

### 3. 为什么语句覆盖不能保证分支覆盖？

```python
def foo(x):
    s = 0           # 语句 1
    if x > 0:       # 语句 2
        s = x       # 语句 3
    # 注意：没有 else
    print(s)        # 语句 4
```

| 测试用例 | x | 语句覆盖 | 分支覆盖 |
|---------|---|---------|---------|
| TC 1: x=1 | 1 | 1,2,3,4 → **100%** ✅ | 只覆盖 True 分支 → **50%** ❌ |

语句全覆盖了，但 `x ≤ 0`（False 分支）从未被测试。

## 五、覆盖强度对比总结

| 特性 | 语句覆盖 | 分支覆盖 |
|------|---------|---------|
| **覆盖目标** | 每条语句至少执行一次 | 每个分支至少执行一次 |
| **强度** | 最弱的覆盖标准 | 比语句覆盖更强 |
| **能否发现缺失 else 的 bug** | ❌ 不能 | ✅ 能 |
| **所需测试用例数** | 较少 | 较多 |
| **包含关系** | 被分支覆盖包含 | 包含语句覆盖 |
| **业界常见要求** | 基础要求 | 通常要求 ≥ 80% |
