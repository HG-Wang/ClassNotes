## 一、数据库：PostgreSQL 16

我们统一使用 PostgreSQL 作为实验数据库。下面提供三种安装方式，优先推荐 Docker。

### 方案一：Docker 安装（首选推荐）

Docker 可以一键拉起一个干净的数据库环境，不污染你的系统，也不会因为操作系统不同而踩坑。

#### 第 1 步：安装 Docker Desktop

| 操作系统 | 下载地址 | 注意事项 |
|---------|---------|---------|
| Windows 10/11 | [Docker Desktop](https://www.docker.com/products/docker-desktop/) | 安装时勾选 WSL2；如果是 Windows Home 旧版无法开启 Hyper-V，请用方案二 |
| macOS | [Docker Desktop](https://www.docker.com/products/docker-desktop/) | Intel 和 Apple 芯片都支持 |
| Linux | [官方安装文档](https://docs.docker.com/engine/install/) | 按发行版选择对应步骤 |

#### 第 2 步：一行命令启动数据库

打开终端（Windows 用 PowerShell），执行：

```bash
docker run --name db-lab \
  -e POSTGRES_USER=student \
  -e POSTGRES_PASSWORD=db2026 \
  -e POSTGRES_DB=labdb \
  -p 5432:5432 \
  -v pgdata:/var/lib/postgresql/data \
  -d postgres:16
```

> 这里的 `-v pgdata:...` 会把数据存到一个持久卷里，**重启电脑或容器都不会丢数据**。

#### 第 3 步：验证是否成功

```bash
docker exec -it db-lab psql -U student -d labdb -c "SELECT version();"
```

看到类似 `PostgreSQL 16.x ...` 的输出就说明一切正常。

#### 日常使用的几条命令

```bash
docker start db-lab      # 开机后启动容器（容器不会自动随系统启动）
docker stop db-lab       # 关闭容器
docker logs db-lab       # 看日志，排查启动报错时有用
docker exec -it db-lab psql -U student -d labdb  # 进入命令行
```

#### 实验 8 专用：带向量扩展的版本

实验 8（向量数据库）需要 pgvector 扩展，用下面的命令另起一个容器：

```bash
docker run --name db-lab-vec \
  -e POSTGRES_USER=student \
  -e POSTGRES_PASSWORD=db2026 \
  -e POSTGRES_DB=vecdb \
  -p 5433:5432 \
  -v pgvecdata:/var/lib/postgresql/data \
  -d pgvector/pgvector:pg16

# 进去开启向量扩展
docker exec -it db-lab-vec psql -U student -d vecdb -c "CREATE EXTENSION vector;"
```

> 端口映射成了 **5433**（不是 5432），两个容器可以同时跑，互不冲突。

### 方案二：直接安装 PostgreSQL（Docker 用不了时的备选）

| 操作系统 | 安装方式 |
|---------|---------|
| Windows | 下载 [EDB 安装包](https://www.enterprisedb.com/downloads/postgres-postgresql-downloads)，选 16 版本，一路下一步，**记住你设置的密码** |
| macOS | 终端执行 `brew install postgresql@16 && brew services start postgresql@16` |
| Linux (Ubuntu) | `sudo apt install postgresql-16` |

装好后进入命令行，创建实验用的账号和数据库：

```sql
CREATE USER student WITH PASSWORD 'db2026';
CREATE DATABASE labdb OWNER student;
GRANT ALL PRIVILEGES ON DATABASE labdb TO student;
```

> 如果后续要做实验 8，pgvector 扩展需要手动编译安装，步骤见 [pgvector GitHub](https://github.com/pgvector/pgvector)。比较麻烦，建议实验 8 时换用 Docker 方案或在线服务。

## 二、数据库客户端

你需要一个图形界面工具来写 SQL 和查看结果。推荐 DBeaver，当然用其他的也行：

| 工具 | 费用 | 一句话点评 |
|------|------|-----------|
| **DBeaver Community** | 免费 | 全平台通用，功能齐全，**课程首选** |
| pgAdmin 4 | 免费 | PostgreSQL 官方出品，装 PG 时自带 |
| DataGrip | 学生免费 | JetBrains 全家桶之一，体验最好，但要申请教育许可 |
| Navicat | 付费 | 界面友好，有 14 天试用 |
| psql 命令行 | 免费 | 最轻量，熟悉命令行的同学可以直接用 |

### DBeaver 连接配置

1. 去 [dbeaver.io](https://dbeaver.io/download/) 下载安装
2. 打开后点左上角"新建连接" → 选择 PostgreSQL
3. 填写：
   - 主机：`localhost`
   - 端口：`5432`（pgvector 容器用 `5433`）
   - 数据库：`labdb`
   - 用户名：`student`
   - 密码：`db2026`
4. 点"测试连接"，显示绿色对勾就 OK 了

## 三、Python 环境（实验 8 和 9 需要）

### 安装 Python

- 版本要求：Python 3.10 及以上
- 推荐用 [Miniconda](https://docs.conda.io/en/latest/miniconda.html) 管理，省去很多依赖问题

### 创建虚拟环境

```bash
# conda 方式
conda create -n dblab python=3.11 -y
conda activate dblab

# 如果不用 conda，也可以用 venv
python -m venv dblab-env
source dblab-env/bin/activate   # macOS / Linux
dblab-env\Scripts\activate      # Windows
```

### 安装依赖

```bash
# 实验 8-9 必装
pip install psycopg2-binary numpy openai streamlit

# 实验 8 选装（自己生成向量，可加分）

```

## 四、国内大模型 API 配置

实验 8、9 需要调用大语言模型。国内主流大模型的 API 都兼容 OpenAI 的接口格式，代码写法完全一样，只需要改两个参数：`base_url` 和 `api_key`。

# Kimi（月之暗面）
# client = OpenAI(api_key="你的key", base_url="https://api.moonshot.cn/v1")
# model = "moonshot-v1-auto"

# 智谱 GLM
# client = OpenAI(api_key="你的key", base_url="https://open.bigmodel.cn/api/paas/v4")
# model = "glm-4-flash"  # 这个模型免费

# 豆包 Seed（字节跳动）
# client = OpenAI(api_key="你的key", base_url="https://ark.cn-beijing.volces.com/api/v3")
# model = "doubao-seed-1-6-250615"

# 通义千问（阿里）
# client = OpenAI(api_key="你的key", base_url="https://dashscope.aliyuncs.com/compatible-mode/v1")

# DeepSeek
# client = OpenAI(api_key="你的key", base_url="https://api.deepseek.com")

## 六、常见问题

### Docker 问题

**Q：Docker Desktop 启动时报"WSL2 not installed"**
A：用管理员权限打开 PowerShell，执行 `wsl --install`，然后重启电脑，再打开 Docker。

**Q：容器跑起来了但连不上数据库**
A：多半是端口被占了。跑一下 `netstat -an | grep 5432` 看看。如果被占用，把命令里的 `-p 5432:5432` 改成 `-p 15432:5432`，然后客户端端口也改成 15432。

**Q：我把容器删了，数据全没了**
A：如果启动时加了 `-v pgdata:...` 参数，数据还在。执行 `docker volume ls` 确认 pgdata 存在，重新用同样的命令建容器就能恢复。如果当时没加 `-v`，那数据确实丢了，只能重新导入。

### 数据库问题

**Q：用 COPY 命令导入 CSV 报"Permission denied"**
A：这是因为 `COPY` 是服务端命令，需要超级用户权限。改用 `\copy`（注意是小写、带反斜杠），这是客户端命令，用 student 用户就能跑。

**Q：中文数据显示乱码**
A：先检查数据库编码：`SHOW server_encoding;`，应该是 `UTF8`。再检查你的 CSV 文件是不是 UTF-8 编码（用记事本或 VS Code 打开看右下角）。

### Python 问题

**Q：`pip install psycopg2-binary` 装不上**
A：试试 `pip install psycopg2-binary --only-binary :all:`；如果还不行，换用 `pip install psycopg`（不带 2）。

**Q：调用大模型 API 超时或报错**
A：上面列的国内平台都不需要翻墙，正常网络就能访问。如果你开了代理，可能反而会干扰，试试关掉代理再跑。
