## 一、成绩怎么算

| 部分 | 占比 | 说明 |
|------|------|------|
| 9 次实验报告 | 90% | 每次 10 分，按下面的维度打分 |
| 综合表现 | 10% | 出勤、课堂参与、进步幅度 |

### 每次实验看什么

| 维度 | 看的是什么 |
|------|-----------|
| **过程完整性** | 检查点有没有逐个做完、截图齐不齐 |
| **技术正确性** | SQL 能不能跑通、结果对不对 |
| **理解深度** | "学习反思"部分是不是用自己的话写的，不是照搬概念 |
| **LLM 使用规范** | 有没有如实填写使用声明 |

> 每次实验的详细评分比例见各实验文档的"考核点"表格。

## 二、实验报告怎么写

### 格式

- **文件类型**：PDF（从 Markdown、Word 或 LaTeX 导出都行）
- **文件命名**：`实验X_学号_姓名.pdf`，例如 `实验3_2023010101_张三.pdf`
- 实验 9 额外交一份代码包：`实验9_学号_姓名_code.zip`

### 每份报告必须包含

1. **基本信息**：姓名、学号、日期、实验名称
2. **检查点截图**：实验文档里标了"检查点 X"的地方，都要有对应截图
3. **SQL / 代码**：用代码块格式粘贴，**不要截图代码**（方便查重和复现）
4. **学习反思**：每次实验最后都有 1-2 个思考题，用自己的话回答
5. **LLM 使用声明**：下面第三部分会详细说

### 截图要求

- 截图要完整——能看到 SQL 语句和返回结果
- 最好带时间信息（DBeaver 底部状态栏、终端时间戳都算）
- 明显拼接或后补的截图不算数

### 提交

- 上传到课程平台（地址见课程群通知）
- 截止时间：**每次实验后一周内**
- 迟交扣分：每迟一天扣 10%，超过 3 天不再接收

## 三、关于 AI / LLM 的使用

### 我们的态度

这门课**允许也鼓励**你用大模型辅助学习——Kimi、GLM、豆包、通义千问、DeepSeek，随便用哪个都行。AI 已经是开发者的日常工具了，学会跟 AI 协作本身就是一种能力。

但有一条底线：**理解原理永远比拿到答案更重要。**

### 推荐的国内大模型

| 模型 | 平台 | 特点 |
|------|------|------|
| ECNU | chat.ecnu.edu.cn | 校内免费部署，额度高 |
| Kimi | moonshot.cn | 中文理解好，擅长长文本 |
| 智谱 GLM-4 | bigmodel.cn | 代码能力强，glm-4-flash 免费 |
| 豆包 Seed | volcengine.com | 字节出品，推理能力不错 |
| 通义千问 | bailian.aliyun.com | 阿里出品，SQL 相关问题表现好 |
| DeepSeek | deepseek.com | 性价比高，推理模型好用 |

不限于以上列表，你用其他的也完全可以。

### 怎么用是鼓励的

- 查语法："PostgreSQL 的 EXTRACT 函数怎么用？"
- 问概念："帮我解释一下 MVCC 是什么原理"
- 调 Bug："这条 SQL 报了 GROUP BY 的错，为什么？"
- 对比学习：实验 2、3 里要求的"和 LLM 比较写法"环节
- 写代码：实验 8、9 里的 Python 脚本
- 要优化建议：实验 5 里的 AI-DBA 环节

### 怎么用是不行的

- **抄了不验证**：直接把 AI 生成的 SQL 贴上去交差，自己没跑过也没看懂
- **该自己想的让 AI 想**：比如实验 4 让你设计 E-R 图，这个得自己画
- **反思部分照抄 AI**："学习反思"问的是你的理解，不是 AI 的理解
- **偷偷用不说**：用了但没在声明里写

### 使用声明怎么写

每份报告最后加一段，格式参考：

```
## LLM 使用声明

本次实验中，我使用了以下 AI 工具辅助：

1. Kimi：阶段 B Q2 —— 查了 NOT EXISTS 和 LEFT JOIN WHERE IS NULL 的区别
2. 通义千问：阶段 C —— 按实验要求做了 Text-to-SQL 对比实验

如果没用，就写"本次实验未使用 LLM 辅助"。
```

简单明了就好，不用写小作文，但要**如实**。

### 每次实验的 AI 开放程度

不是每次实验都一样的——前几次要求多动手，后面逐步放开：

| 实验 | 开放度 | 怎么理解 |
|------|--------|---------|
| 实验 1 | 低 | 只能查语法，建表语句必须自己写 |
| 实验 2 | 中 | 有 LLM 对比环节，但查询得先自己写出来 |
| 实验 3 | 中 | 有 Text-to-SQL 对比，同样要求先手写 |
| 实验 4 | 低 | E-R 图、规范化分析必须独立完成 |
| 实验 5 | 中 | 有 AI-DBA 环节，但要对 AI 建议做批判分析 |
| 实验 6 | 低 | 并发实验必须自己在两个窗口里一步步操作 |
| 实验 7 | 中 | 有 LLM 辅助的模型对比讨论 |
| 实验 8 | 中 | 向量加载可以用脚本，但对比实验要亲手跑 |
| 实验 9 | 高 | 本身就是 AI 集成实验，但核心逻辑要自己写 |

## 四、学术诚信

### 什么算抄袭

- 两个人的报告 SQL / 代码几乎一模一样
- 截图造假（P 图改结果、用别人的截图）
- 用了 AI 但不写声明，装作是自己写的
- 提交了 AI 编造的代码（比如查了一张数据库里根本不存在的表）——这叫"AI 幻觉"，你没验证就交上来，算你的责任

### 怎么处理

- 第一次：该次实验记 0 分，单独谈话提醒
- 再犯：课程总评不及格

### 鼓励的协作

- 互相讨论思路、概念，完全没问题
- 帮同学解决环境配置问题，也没问题
- **但 SQL / 代码 / 报告必须各写各的**
